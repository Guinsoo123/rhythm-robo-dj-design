# 人形机器人强化学习预备修

本文面向准备阅读本项目算法设计的人。目标不是把数学和强化学习讲成教材，而是建立一套能直接服务于“人形机器人随音乐稳定跳舞”的基础语言。

阅读完本文后，应能理解这些问题：

- 为什么机器人控制要写成状态、动作、奖励和策略。
- 为什么相位、概率分布、梯度、价值函数会出现在舞蹈控制里。
- 每个数学概念在机器人语境中到底对应什么传感器、控制量或训练信号。

## 项目问题的一句话形式化

本项目希望训练一个策略：

```latex
\pi_\theta(a_t \mid s_t)
```

它在每个控制时刻读取机器人状态与音乐条件 `s_t`，输出动作 `a_t`，再由仿真器或真机控制器执行。训练目标是让机器人在不摔倒、不越界的前提下，动作与摇滚音乐的节拍、能量和段落相匹配。

## 数学基础概念

| 概念 | 定义 | 直觉 | 机器人语境 |
| --- | --- | --- | --- |
| 标量 scalar | 单个数值，记作 `x` | 一个温度、一个速度、一个分数 | 奖励 `r_t`、BPM、机身高度、单个关节角 |
| 向量 vector | 有序数列，记作 `x in R^n` | 把多个相关数打包 | 全身关节角 `q`、关节速度 `qdot`、观测向量 `s_t` |
| 矩阵 matrix | 二维数表，记作 `A in R^{m x n}` | 描述多个输入如何线性混合成多个输出 | 神经网络权重、雅可比矩阵、状态归一化统计 |
| 张量 tensor | 多维数组 | 批量数据的容器 | 并行环境 rollout：`[time, env, feature]` |
| 范数 norm | 向量长度，如 `||x||_2` | 距离原点有多远 | 动作幅度、速度惩罚、姿态误差大小 |
| 内积 dot product | `x^T y` | 两个方向是否一致 | 速度方向与目标方向对齐、足端力与接触法向关系 |
| 角度 angle | 两个方向之间的夹角 | 朝向差了多少 | 机身朝向、头部摆动相位、重力方向投影 |
| 正弦/余弦 | 周期函数 `sin(phi), cos(phi)` | 用连续方式表示循环 | beat phase 不能只用 0/1 脉冲，应编码为 `[sin(phi), cos(phi)]` |
| 概率 probability | 事件发生可能性，取值 `[0,1]` | 不确定性的数值表达 | 策略不是死板输出，可用高斯分布探索动作 |
| 随机变量 random variable | 结果不固定的变量 | 每次试验可能不同 | 随机初始姿态、随机摩擦、策略采样动作 |
| 概率分布 distribution | 随机变量取值规律 | “大多在哪里，偶尔去哪里” | 高斯策略给出动作均值和方差 |
| 期望 expectation | 分布下的平均值 `E[X]` | 多次尝试后的平均表现 | 强化学习最大化期望回报，而不是只看一次成功 |
| 方差 variance | 偏离平均值的程度 | 稳不稳定、波动大不大 | 动作噪声、回报波动、训练是否发散 |
| 条件概率 | 已知条件下的概率 `P(A|B)` | 看见当前状态后再判断下一步 | 动力学 `P(s_{t+1}|s_t,a_t)` |
| 梯度 gradient | 函数对参数变化的方向导数 `nabla f` | 往哪个方向改能让目标变大或变小 | 用 `nabla_theta J(theta)` 更新策略网络 |
| 雅可比 Jacobian | 多输出对多输入的一阶导数组成的矩阵 | 每个输入影响每个输出多少 | 足端速度对关节速度的关系、动作对奖励的局部影响 |
| 海森矩阵 Hessian | 二阶导数组成的矩阵 | 曲面弯曲程度 | 高级优化和稳定性分析中判断目标函数曲率 |
| 优化 optimization | 寻找让目标最大或最小的变量 | 调旋钮直到效果最好 | 调整策略参数，让长期奖励最大 |
| 约束 constraint | 变量必须满足的条件 | 有些旋钮不能越界 | 关节限位、力矩上限、机身倾角、安全速度 |
| 离散时间 | 时间按步编号 `t=0,1,2,...` | 视频帧或控制帧 | 每 10 ms 更新一次策略就是 100 Hz 控制 |
| 连续时间 | 时间连续变化 `t in R` | 真实物理世界的时间 | MuJoCo 在更小步长内积分动力学 |

## 机器人运动学与动力学概念

| 概念 | 定义 | 直觉 | 本项目相关解释 |
| --- | --- | --- | --- |
| 自由度 DOF | 系统可独立运动的变量数量 | 有多少个可控关节或方向 | 人形机器人腿、腰、臂、头部各有关节自由度 |
| 关节角 `q` | 每个关节的位置变量 | 膝盖弯了多少、髋关节转了多少 | 常作为观测输入，也可作为位置控制目标 |
| 关节速度 `qdot` | 关节角随时间变化率 | 关节动得多快 | 速度过大容易抖动或伤机，需要惩罚和限幅 |
| 关节力矩 `tau` | 电机对关节施加的转矩 | 推动关节转动的“劲” | 高级控制可直接输出力矩，第一版更建议输出位置残差 |
| 位姿 pose | 位置和朝向的组合 | 机器人在哪里、朝哪边 | 机身姿态用于判断是否摔倒 |
| 四元数 quaternion | 表示三维旋转的四个数 | 避免欧拉角万向节锁 | IMU 常输出四元数或可转换为重力方向 |
| 重心 CoM | 质量分布的平均位置 | 身体重量“集中点” | 重心越出支撑区域越容易摔倒 |
| 支撑多边形 | 接触点在地面形成的支撑区域 | 脚踩出来的安全范围 | 人形机器人跳舞时重心应尽量落在支撑区域附近 |
| 接触 contact | 足端与地面发生约束和力交换 | 脚有没有踩实地面 | 奖励和安全层要监控脚底接触、滑移和冲击 |
| 摩擦 friction | 接触面抵抗滑动的能力 | 地面滑不滑 | 训练时随机化摩擦，提高 sim-to-real 鲁棒性 |
| 低层控制器 | 高频执行关节目标的控制回路 | 策略负责节奏，底层负责跟踪 | 常用 PD 控制或机器人厂商 SDK 内环 |
| PD 控制 | `tau = Kp(q* - q) - Kd qdot` | 偏差越大拉回越强，速度越快阻尼越强 | RL 输出 `q*`，PD 把目标转成电机控制 |
| 动作残差 | 在默认站姿上叠加小动作 `q* = q_default + alpha a` | 先站稳，再摇摆 | 降低学习难度，避免一开始输出危险姿态 |
| 动力学 | 力、质量、惯性决定运动变化的规律 | 推一下会怎么动 | MuJoCo 用动力学从 `s_t,a_t` 推进到 `s_{t+1}` |
| 稳定性 | 扰动后能否保持或恢复可控姿态 | 晃了但不倒 | 跳舞动作必须在稳定性边界内表达风格 |

## 强化学习核心概念

| 概念 | 定义 | 直觉 | 机器人语境 |
| --- | --- | --- | --- |
| 智能体 agent | 做决策的主体 | 机器人“大脑” | 策略网络和训练算法 |
| 环境 environment | 接收动作并返回新状态与奖励的系统 | 机器人所处世界 | MuJoCo 仿真器或真机加地面和音乐 |
| 状态 state | 决策所需信息的完整描述 | 当前局面 | 理想上包含机器人真实物理状态和音乐相位 |
| 观测 observation | 智能体实际能看到的信息 | 摄像头/IMU/编码器给到的局面 | 实际训练中常把观测也记作 `s_t` |
| 动作 action | 智能体输出的控制变量 | 下一步怎么动 | 关节位置残差、速度目标或力矩 |
| 奖励 reward | 对一步行为的标量评价 | 做得好加分，危险扣分 | 稳定、跟拍、风格、省力、安全共同组成 |
| 策略 policy | 从状态到动作分布的函数 | 看局面做决定 | `pi_theta(a_t|s_t)` 输出动作均值和噪声 |
| 回报 return | 从当前开始累计的折扣奖励 | 长期总成绩 | 不能只奖励当前跟拍，还要考虑后续会不会摔 |
| 折扣因子 `gamma` | 未来奖励的衰减系数 | 近处更重要，远处仍有价值 | 常取 0.95 到 0.995，机器人控制常用较高值 |
| 价值函数 `V(s)` | 从状态 `s` 出发的期望回报 | 这个局面有多好 | 站稳且接近节拍相位的状态价值更高 |
| 动作价值 `Q(s,a)` | 在状态 `s` 采取动作 `a` 后的期望回报 | 这个动作在这个局面有多好 | 某个大幅摆动可能当前好看但后续会失衡 |
| 优势函数 `A(s,a)` | 动作比平均策略好多少 | 这个动作是否超出预期 | PPO 用优势决定加强或削弱该动作概率 |
| MDP | 马尔可夫决策过程，由状态、动作、转移、奖励组成 | 当前局面足够决定下一步概率 | 机器人控制通常近似为 MDP |
| 马尔可夫性 | 下一步只依赖当前状态和动作 | 不必记完整历史 | 若观测缺少速度或历史动作，马尔可夫性会变差 |
| rollout | 策略与环境交互得到的一段轨迹 | 机器人试跳一段 | PPO 用多环境 rollout 更新网络 |
| episode | 从 reset 到终止的一次完整尝试 | 一段训练舞蹈 | 摔倒、超时或歌曲片段结束都可终止 |
| on-policy | 用当前策略采集的数据更新当前策略 | 边试边学当前做法 | PPO 属于 on-policy，数据新鲜但样本消耗大 |
| off-policy | 可用旧策略或回放数据学习 | 旧经验也能用 | SAC/TD3 常见，连续控制样本效率高 |
| actor-critic | 同时学习策略 actor 和价值 critic | 一个负责做，一个负责评价 | PPO 常用 actor 输出动作，critic 估计 `V(s)` |
| 熵 entropy | 分布不确定性大小 | 探索有多大胆 | 训练早期鼓励动作多样，后期逐步收敛 |
| PPO | 近端策略优化 | 小步更新，避免策略突然变坏 | 本项目第一版优先选择 PPO，稳定且工程成熟 |
| GAE | 广义优势估计 | 在偏差和方差之间折中估计优势 | 让机器人训练曲线更平滑 |
| 域随机化 | 训练时随机改变环境参数 | 让策略见过更多“世界” | 随机质量、摩擦、延迟、传感噪声 |
| 课程学习 | 从简单任务逐渐加难 | 先会站稳，再会小幅跟拍，再会风格动作 | 降低人形机器人从零学习的难度 |
| sim-to-real | 从仿真迁移到真机 | 先在虚拟世界练熟再上真机 | 需要随机化、安全层、动作缩放和真机小幅验证 |

## 音乐条件概念

| 概念 | 定义 | 直觉 | 机器人语境 |
| --- | --- | --- | --- |
| BPM | 每分钟节拍数 | 歌有多快 | 控制动作周期，不直接决定动作幅度 |
| beat | 音乐中稳定的拍点 | 跟着点头的“点” | 动作相位应在 beat 附近形成重音 |
| downbeat | 小节第一拍 | 更强的节奏落点 | 可以触发更明显的身体下沉或头部重音 |
| onset | 声音能量突然出现的时刻 | 鼓点、吉他拨弦的攻击 | 可用于强化动作瞬间感 |
| RMS energy | 音频短窗能量 | 这一刻响不响、强不强 | 映射到动作幅度上限或奖励权重 |
| beat phase | 当前处在一个节拍周期的哪个位置 | 拍内进度条 | 用 `sin/cos` 编码，让策略知道循环位置 |
| section | 歌曲段落，如主歌、副歌 | 音乐气氛区间 | 副歌可允许更大动作，间奏可减小动作 |

## 本项目的最小心智模型

可以把训练过程想成：

```text
音乐特征 + 机器人传感
  -> 观测 s_t
  -> 策略 pi_theta 采样动作 a_t
  -> 安全缩放与低层控制
  -> MuJoCo 推进物理
  -> 计算稳定、节奏、风格、安全奖励
  -> PPO 更新策略
```

关键不是让机器人“背一套舞步”，而是让它学会一个条件控制函数：同样的策略在不同 BPM、不同音乐能量、不同姿态扰动下，都能选择相对安全且有节奏感的动作。

## 推荐先掌握的优先级

| 优先级 | 需要掌握 | 原因 |
| --- | --- | --- |
| P0 | 状态、动作、奖励、策略、回报 | 这是读懂所有 RL 文档的入口 |
| P0 | 关节角、关节速度、PD 控制、接触 | 这是读懂机器人控制接口的入口 |
| P1 | 期望、概率分布、梯度、优势函数 | 这是读懂 PPO 的入口 |
| P1 | beat phase、BPM、onset、energy | 这是读懂音乐条件输入的入口 |
| P2 | Bellman 方程、策略梯度、GAE 推导 | 这是调试训练稳定性的入口 |
| P2 | 域随机化、课程学习、sim-to-real | 这是从仿真走向真机的入口 |
