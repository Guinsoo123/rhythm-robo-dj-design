# 研发里程碑

## 阶段 0：文档和同步基础

目标：让设计文档可维护、可同步、可持续迭代。

- 使用 Markdown 编写所有设计文档。
- push `main` 后自动同步到 Notion。
- 明确算法目标、软件架构、MuJoCo/Conda 技术路线。

验收：

- GitHub Actions 可手动运行。
- Notion 父页面出现同步后的文档页面。

## 阶段 1：音乐特征离线管线

目标：把摇滚音乐转换成可训练的条件信号。

- 收集 10 到 30 首不同 BPM 的摇滚片段。
- 使用 librosa 提取 BPM、beat times、onset strength、RMS energy。
- 把特征缓存为 `.npz` 或 `.json`。
- 画出 beat 和能量曲线，人工检查是否合理。

验收：

- 任意输入一首歌，能输出标准化音乐条件序列。
- beat 可视化和听感基本一致。

## 阶段 2：MuJoCo 机器人环境

目标：在 MuJoCo 中稳定加载宇树机器人模型，并建立 Gymnasium 风格环境。

- 导入宇树机器人 MJCF/XML 模型。
- 实现 reset、step、observation、reward、done。
- 实现默认站姿和位置残差控制。
- 实现仿真视频录制。

验收：

- 环境可以连续 step 10000 步不崩溃。
- 零动作时机器人能保持默认站姿。
- 随机小动作不会立即摔倒。

## 阶段 3：固定节拍小幅动作

目标：不使用真实音乐，先让机器人跟随简单相位信号做小幅律动。

- 输入固定 BPM 的 beat phase。
- 训练小幅点头或身体摆动。
- 奖励稳定、节拍对齐、动作平滑。

验收：

- 仿真视频中能看到和节拍一致的周期动作。
- 摔倒率低于 5%。
- 关节角和速度不越界。

## 阶段 4：真实音乐条件训练

目标：使用真实摇滚音乐特征驱动动作。

- 从缓存中随机抽取音乐片段。
- 训练策略适配不同 BPM 和音乐强度。
- 增加能量匹配和 downbeat 强调奖励。
- 加入 beat 噪声和动力学随机化。

验收：

- 未见过的歌曲上仍能保持节奏感。
- beat alignment error 在可接受范围内。
- 音乐强弱变化和动作幅度有相关性。

## 阶段 5：观赏性和鲁棒性增强

目标：让动作更像摇滚表演，而不是普通机械周期运动。

- 设计低维参考动作，例如 headbang、sway、bounce。
- 加入 imitation reward 或 motion prior。
- 加入随机推扰、摩擦变化、控制延迟。
- 批量生成视频进行人工评分。

验收：

- 人工评分中“节奏感”和“摇滚风格”明显提升。
- 随机扰动下摔倒率保持低水平。

## 阶段 6：真机小幅验证

目标：在宇树机器人上验证安全的小幅音乐响应。

- 导出策略为推理格式。
- 动作强度缩放到训练幅度的 20% 到 40%。
- 启用低通滤波、限幅、姿态保护和急停。
- 先无音乐回放动作，再接入音乐特征。

验收：

- 真机可以稳定完成点头/摆动。
- 急停有效。
- 没有关节限位、过速和明显打滑。

## 阶段 7：真机增强演示

目标：在监护环境中展示完整摇滚律动。

- 增加动作幅度。
- 加入轻微下蹲和 bounce。
- 支持不同歌曲切换。
- 输出演示视频和技术复盘。

验收：

- 机器人能随多首摇滚音乐稳定表演。
- 演示前后有完整日志，可追踪每次异常。
