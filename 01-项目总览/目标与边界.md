# 目标与边界

## 项目目标

本项目希望实现一个“Rhythm Robo DJ”：机器人能够听一段摇滚音乐，在保持自身稳定和硬件安全的前提下，做出符合节奏感的摇头、身体摆动、下蹲起跳和重拍强调动作。

这里的“跟着音乐动”包含三层含义：

| 层次 | 目标 | 例子 |
| --- | --- | --- |
| 节拍对齐 | 动作周期和音乐 beat 对齐 | 每个重拍点头一次，或每两拍蹦跳一次 |
| 强弱响应 | 动作幅度随音乐能量变化 | 鼓点密集时身体摆动更明显，安静段落动作收敛 |
| 段落风格 | 不同音乐段落切换动作模式 | 主歌轻微摇头，副歌加入弹跳，breakdown 降低重心 |

## 机器人形态假设

本文档默认使用宇树机器人作为目标平台。实际型号可以是四足机器人或带更多上身自由度的机器人，但算法设计保持通用。

| 能力 | 设计假设 |
| --- | --- |
| 关节控制 | 可发送关节位置、速度或力矩目标；底层控制器负责电机闭环 |
| 状态读取 | 可读取 IMU、关节角、关节速度、足端接触、机身姿态等信息 |
| 运动空间 | 可在原地完成头部/机身摆动、下蹲、轻微弹跳，不要求大范围平移 |
| 安全约束 | 必须限制关节角、速度、力矩、机身倾角、足端打滑和起跳高度 |

## 明确不做什么

| 不做 | 原因 |
| --- | --- |
| 不直接用音乐幅度控制关节 | 音乐信号噪声大，直接映射到关节容易产生危险动作 |
| 不一开始追求真机大幅跳跃 | 真机冲击、失足和电机过载风险高，需要从仿真和小幅动作开始 |
| 不把舞蹈动作写成固定脚本 | 固定脚本难适配不同歌曲，也难处理机器人状态扰动 |
| 不绕过底层安全控制 | 任何学习策略都有探索误差，必须有安全门控和动作裁剪 |

## 成功标准

| 阶段 | 验收标准 |
| --- | --- |
| MuJoCo 可视化 demo | 给定一段摇滚音乐，机器人在仿真中原地稳定摇摆，动作节奏和 beat 可肉眼对应 |
| 强化学习训练 | 策略在随机初始姿态、不同 BPM、不同摩擦系数下都能保持不摔倒 |
| 音乐泛化 | 未参与训练的歌曲也能产生有节奏的动作，不只是记住固定音频 |
| 真机小幅验证 | 真机以低幅度、低速度执行点头/摆动动作，姿态安全，急停有效 |
| 真机增强验证 | 在人工监护下加入轻微屈伸和蹦跳感，但不追求离地大跳 |

## 风险清单

| 风险 | 表现 | 缓解策略 |
| --- | --- | --- |
| 训练策略学会危险动作 | 为了奖励而过度跳跃或扭转 | 奖励加入姿态、力矩、接触冲击惩罚，动作输出加安全裁剪 |
| 音乐节拍识别错误 | 动作和音乐错拍 | 使用 beat tracker + 能量峰值 + 相位平滑，不让单帧检测直接决定动作 |
| sim-to-real 差异 | 仿真很稳，真机发抖或摔倒 | 域随机化、低通滤波、动作幅度逐步放大、真机数据回放评估 |
| 动作缺少观赏性 | 稳定但像机械抖动 | 增加风格奖励、参考动作库、动作平滑和段落级动作切换 |

## 最小可行产品

MVP 不要求机器人真的离地跳跃。第一版只需要：

- 输入一段摇滚音乐文件。
- 自动提取 BPM、beat 时间点、能量曲线和段落强弱。
- 在 MuJoCo 中加载宇树机器人模型。
- 训练或运行一个策略，使机器人原地稳定完成“点头 + 躯干左右摆动 + 膝关节轻微弹跳”。
- 输出仿真视频、节拍对齐曲线和稳定性指标。
